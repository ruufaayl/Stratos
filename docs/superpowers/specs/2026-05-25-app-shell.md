# Stratos — App Shell (sub-project C)

> **Status:** Locked 2026-05-25.
> **Inputs:** brand identity spec, design system v2, information architecture, competitive study.
> **Outputs:** the **chrome that every `/app/[org]/*` route lives inside**. After this ships, no app screen needs to re-derive nav, org context, cloud context, or keyboard wiring.
> **Audience:** the implementer of every subsequent screen.

---

## 1. What this is (and what it is not)

**Is:** the layout primitives, the providers, the routing scaffolding, the global keyboard surface. The "operating system" of the app.

**Is not:** any specific business screen (overview, findings, settings — those are sub-projects D, E, F…). The shell ships with a single placeholder content area that says "Shell ready — plug in routes."

---

## 2. Locked decisions

| # | Decision | Why |
|---|---|---|
| 1 | **⌘K hand-rolled on Radix Dialog** | We already own the Modal primitive on Radix. No new deps. Full brand control from day 1. Trade-off: ~1 extra week vs. cmdk, but exact brand fit. |
| 2 | **No notification bell in v1** | IA showed it; we ship the surface only when we have something to notify about. Cleaner shell, smaller test surface, no retrofit needed (slot is documented). |
| 3 | **Rail: user-toggleable collapse with persisted preference** | Pin/unpin button. Default is **collapsed** for new users (engineers prefer dense). Preference stored per-user in Clerk `publicMetadata.railCollapsed`. |
| 4 | **Org slug in URL is source of truth** | Switching org changes the URL. Never silently rewrites the active org from a dropdown. Deep-linkable. |
| 5 | **Cloud is per-org preference, not in URL** | The URL path implies cloud only when cloud-scoped (`/app/[org]/aws/...`). The active cloud for cloud-scoped sections is held in `localStorage.stratos.activeCloud.<orgId>`. |
| 6 | **`<MotionConfig reducedMotion="user">` is already in place** | Shell animations honour `prefers-reduced-motion: reduce` automatically. |
| 7 | **No tooltips on every icon — only where the meaning is ambiguous** | Section names are not ambiguous to our audience. Tooltips add noise. The collapsed rail still shows labels on hover via the section's own native tooltip-on-hover (Radix Tooltip, 500ms delay). |

## 5. Component contracts

### 5.1 `<AppShell>` (composition root)

```tsx
type AppShellProps = {
  children: React.ReactNode;
};

// Mounts: OrgProvider, CloudProvider, RailProvider, ShortcutsProvider, CommandBarProvider
// Renders: <Rail /> <TopBar /> <main>{children}</main>
```

- Reads initial rail state from server (`Clerk.publicMetadata.railCollapsed`) — falls back to `true` (collapsed) for new users.
- Reads initial cloud from `localStorage.stratos.activeCloud.${orgId}` on mount — falls back to org's only connected cloud, or "aws" if multiple.
- Wraps content in `<MotionConfig reducedMotion="user">` (already done at the root layout — does not double-wrap here).

### 5.2 `<Rail>`

Props: `{ collapsed: boolean }` (controlled — `RailContext` is source of truth)

Behavior:
- Renders items from `lib/shell/nav-registry.ts`.
- Active item: matched against `pathname` via longest-prefix. `Overview` matches `/app/[org]` exactly OR `/app/[org]?tab=*`. `Inventory` matches `/app/[org]/{aws|azure|gcp}/*`. Settings matches `/app/[org]/settings/*`.
- Collapsed → 64px wide, icons only, item gets a Radix Tooltip on hover (500ms delay).
- Expanded → 220px wide, icon + label.
- Active item: 2px `intel-500` left border + bg-bg-elevated.
- Hover (non-active): bg-bg-elevated/50.
- Transition: width animates with `duration.reveal` (320ms) via Framer Motion `<motion.aside layout>`.

### 5.3 `<RailItem>`

```tsx
type RailItemProps = {
  href: string;
  icon: React.ComponentType<{ className?: string }>;   // Lucide icon
  label: string;
  active?: boolean;
  collapsed?: boolean;
  badge?: number;        // optional count (e.g. open-findings count) — defer to wave 2
};
```

Renders as `<Link>` from `next/link`. ARIA: `aria-current="page"` when active.

### 5.4 `<RailToggle>`

Pin/unpin button. Bottom of rail. Icon: `PanelLeftClose` (collapsed) / `PanelLeftOpen` (expanded) from Lucide.

On click:
1. Flip local `railCollapsed` in context.
2. Fire-and-forget POST `/api/me/preferences` with `{ railCollapsed: boolean }` to persist to Clerk publicMetadata.
3. Show no toast — silent preference change.

Keyboard: shortcut `[` toggles. Listed in `lib/shell/shortcuts.ts`.

### 5.5 `<TopBar>`

Layout (left to right):
```
[OrgSwitcher]  ›  [Breadcrumbs]              [⌘K trigger]  [CloudSwitcher]  [UserMenu]
```

Height: 60px. `border-b border-border-subtle`. Background: `bg-bg-canvas` (no contrast against main — clean).

### 5.6 `<OrgSwitcher>`

Radix `DropdownMenu`.

Trigger:
- Collapsed: icon-style "org sigil" (a small colored square derived from `hash(orgId) % palette`) + active-org slug.
- Expanded: same + chevron.

Menu contents:
- Searchable list of user's orgs (Clerk `useOrganizationList`).
  - Each row: sigil + name + slug (mono, dim).
  - Clicking → `router.push("/app/<otherOrgSlug>")` — preserves the current section path if possible (e.g. switching from `/app/acme/findings` → `/app/contoso/findings`), else falls back to org root.
- Divider.
- "Create org" → `router.push("/orgs/create")`.
- "Manage orgs" → `router.push("/orgs")`.

When user has exactly 1 org and 0 invites: trigger is **not interactive** (still shows current org, but click is no-op). The dropdown contents are only shown when there is something to switch to.

### 5.7 `<CloudSwitcher>`

Radix `DropdownMenu`. Only rendered when:
- The current route is cloud-scoped (matches `/app/[org]/(aws|azure|gcp)/...`)
- AND the org has 2+ connected clouds.

Otherwise: not rendered.

Menu:
- `AWS` `Azure` `GCP` (only those connected for this org).
- Current cloud highlighted with `intel-500` checkmark.
- Clicking → `router.push` to the same section in the target cloud (e.g. `/app/acme/aws/services/ec2` → `/app/acme/azure/services/compute`). If the target section doesn't exist in that cloud, fall back to the cloud root (`/app/acme/azure`).
- AND updates `localStorage.stratos.activeCloud.${orgId}` so cloud-agnostic sections honour the preference.

### 5.8 `<UserMenu>`

Radix `DropdownMenu`. Trigger: user avatar (Clerk `<UserButton />` wrapped so we can intercept the menu).

Menu items (top to bottom):
- User name + email (non-interactive header)
- Divider
- Profile → `/me`
- Security → `/me/security`
- Divider
- Theme → submenu (Dark — only option at v1, disabled placeholder for Light/System)
- Keyboard shortcuts → opens `<KeyboardShortcutsDialog>` (Modal listing all shortcuts from `lib/shell/shortcuts.ts`)
- Help → external link to docs root in new tab
- Divider
- Admin (only if `user.publicMetadata.staff === true`) → `/admin`
- Sign out → Clerk `signOut()` → redirect `/`

### 5.9 `<Breadcrumbs>`

Derives from `usePathname()`. Maps each segment to a human label via `lib/shell/breadcrumb-registry.ts`. Renders up to 3 segments. Last segment is non-clickable (current page). Separator: `›` in `text-text-muted`.

Special cases:
- `/app/[org]` root → no breadcrumbs (just shows section name "Overview" handled by topbar)
- Detail pages (`.../findings/[id]`) → segment label is the entity's display name, resolved by an opt-in `useBreadcrumbTitle("Reduce idle: i-0abc1234")` hook the leaf page can call.

### 5.10 `<CommandBar>` (⌘K)

Built on the existing `Modal` primitive (Radix Dialog under the hood).

Visual:
```
┌──────────────────────────────────────────────────────┐
│ [🔍] Type a command, search resources, or jump...   │ Esc
├──────────────────────────────────────────────────────┤
│ RECENT                                               │
│   ▸ Findings: idle EC2                               │
│   ▸ Settings → Members                               │
│                                                      │
│ NAVIGATE                                             │
│   ▸ Overview                       ⌘O                │
│   ▸ Findings                       ⌘F                │
│   ▸ Forecast                                         │
│   ▸ Settings → Billing                               │
│                                                      │
│ ACTIONS                                              │
│   ▸ Invite member                                    │
│   ▸ Connect AWS account                              │
│                                                      │
│ SAVED                                                │
│   ▸ "Idle EC2 us-east-1" (saved filter)              │
│                                                      │
│ RESOURCES (4)                                        │
│   ▸ i-0abc1234ef · t3.xlarge · acme-prod             │
│   ▸ bucket logs-archive                              │
└──────────────────────────────────────────────────────┘
   ↑↓ navigate     ↵ open     Esc dismiss
```

Behavior:
- Opens on `⌘K` (mac) / `Ctrl+K` (others). Also via the trigger button in TopBar.
- Input focused on open.
- Fuzzy search across ALL indexed objects (see §5.11). Source-of-truth ranking: recency × score.
- Categories rendered in this fixed order: **Recent** (last 5) → **Navigate** → **Actions** → **Saved** → **Resources** → **Findings** → **Help**. Empty categories are not rendered.
- Each category shows up to 5 results initially; "Show more" link if there are more.
- Keyboard: `↑/↓` cycles, `↵` opens, `⌘↵` opens in new tab (for navigate items), `Esc` closes. `Tab` does nothing (steal-and-trap inside palette so focus doesn't jump).
- On open, restores caret position to last query for the session. On close, query is cleared after 200ms (so reopening within that window restores; opening fresh starts blank).

Accessibility:
- Modal already provides focus trap + ESC + accessible name.
- Input has `aria-label="Command bar — type to search"`.
- Each result is a button with `aria-selected` when keyboard-focused.
- Results region is `role="listbox"`, items are `role="option"`. Live region announces selection changes.

### 5.11 `<CommandBarSource>` (index builder)

The index is **assembled at command-bar-open time** to keep it fresh. It is NOT a static global. The build cost is acceptable because the index is bounded (~200 items).

Sources:
1. **Recent** (max 5) — from `localStorage.stratos.commandBar.recent.${orgId}` (latest first).
2. **Navigate** — from `lib/shell/nav-registry.ts`, expanded to include sub-pages from `lib/shell/breadcrumb-registry.ts`. ~30 items.
3. **Actions** — from a hand-curated list in `lib/shell/actions-registry.ts` (e.g. "Invite member", "Connect AWS account", "Create report", "Switch org → …", "Sign out"). Each action specifies required role.
4. **Saved** — fetched lazily from `/api/saved-views?org=X` on open. Cached for 60s.
5. **Resources** — fetched lazily from `/api/resources/search?q=X&org=X` ONLY when the query is ≥ 3 chars. Debounced 200ms.
6. **Findings** — fetched lazily from `/api/findings/search?q=X&org=X` when query ≥ 3 chars. Debounced 200ms.
7. **Help** — from a hand-curated list pointing to `/docs/*` pages. Hardcoded.

Ranking:
- Each result has a `score` ∈ [0, 1] from the fuzzy match.
- Final = `score * (1 + recencyBoost)`. RecencyBoost = 0.5 if item is in `Recent`, else 0.
- Ties broken by category order (Recent first).

### 5.12 `<Shortcuts>` (global keyboard)

Mounted once at `AppShell`. Reads from `lib/shell/shortcuts.ts`:

```ts
export const shortcuts = [
  { keys: ["mod", "k"],   label: "Open command bar",  scope: "global",   action: openCommandBar },
  { keys: ["mod", "/"],   label: "Open command bar",  scope: "global",   action: openCommandBar }, // duplicate (cmdk pattern)
  { keys: ["g", "o"],     label: "Go to Overview",    scope: "app",      action: goto("/") },
  { keys: ["g", "f"],     label: "Go to Findings",    scope: "app",      action: goto("/findings") },
  { keys: ["g", "r"],     label: "Go to Forecast",    scope: "app",      action: goto("/forecast") },
  { keys: ["g", "s"],     label: "Go to Settings",    scope: "app",      action: goto("/settings") },
  { keys: ["["],          label: "Toggle nav rail",   scope: "app",      action: toggleRail },
  { keys: ["?"],          label: "Show shortcuts",    scope: "app",      action: openShortcutsDialog },
  { keys: ["Escape"],     label: "Dismiss overlay",   scope: "modal",    action: dismissTopOverlay },
] as const;
```

`mod` is `⌘` on mac, `Ctrl` elsewhere. Two-key sequences (`g o`) use a 1.5-second window between keys. Disabled when an input/textarea is focused (except the command bar itself, which handles its own keys).

## 6. Providers + context contracts

### 6.1 `<OrgProvider>`

```tsx
type OrgValue = {
  org: { id: string; slug: string; name: string; sigilColor: string };
  role: "owner" | "admin" | "member";
  switchTo: (slug: string) => Promise<void>;     // delegates to router + Clerk
};
const OrgContext = createContext<OrgValue | null>(null);
export const useOrg = () => { /* throws if outside provider */ };
```

Provider derives `org` from URL slug (`useParams().org`) + Clerk's active organization. Throws a hard error if URL slug ≠ Clerk active org (impossible in middleware-correct app, but guards against drift).

### 6.2 `<CloudProvider>`

```tsx
type Cloud = "aws" | "azure" | "gcp";
type CloudValue = {
  active: Cloud;
  available: readonly Cloud[];
  setActive: (c: Cloud) => void;
};
```

`available` is fetched from `/api/orgs/{orgId}/clouds` once per org session.
`active` is initialized from URL > localStorage > first available.
`setActive` updates localStorage AND routes if necessary (see §5.7).

### 6.3 `<RailProvider>`

```tsx
type RailValue = { collapsed: boolean; toggle: () => void; setCollapsed: (v: boolean) => void };
```

Persists on toggle via debounced POST `/api/me/preferences` (300ms debounce — user can spam pin/unpin without flooding server).

### 6.4 `<CommandBarProvider>`

Owns: `open: boolean`, `query: string`, `selectedIndex: number`.
Methods: `openCommandBar()`, `closeCommandBar()`, `setQuery(s)`.

## 7. Routing + middleware

### 7.1 Middleware (extend existing `apps/web/middleware.ts`)

Order of checks:
1. **Public routes** pass through (`/`, `/proof`, `/pricing`, `/sign-in`, `/sign-up`, `/legal/*`, `/docs/*`, `/_next/*`, `/api/health`).
2. **Authed route, not signed in** → redirect `/sign-in?return_to=<path>`.
3. **`/app/[org]/*` route**:
   a. Resolve `org` slug → org ID via Clerk org list. If user is not a member of `org`, redirect to `/orgs?error=not-member`.
   b. Ensure Clerk's active organization matches URL slug. If not, set active org server-side via Clerk API and continue.
   c. Check user has at least one connected cloud for this org. If not, redirect `/app/[org]/welcome`.

Edge cases:
- `/app/[org]/welcome` is the **only** `/app/[org]/*` route that does NOT require a connected cloud (it's the wizard).
- `/app/[org]/settings/billing/danger` requires `role=owner` — middleware checks; otherwise renders `<Empty kind="permission-denied" />` server-side.

### 7.2 Layouts

- `app/layout.tsx` — root, already exists, mounts `<MotionConfig>` + `<ClerkProvider>` + global font vars. **Unchanged.**
- `app/app/[org]/layout.tsx` — new. Server component. Resolves `org` from URL + Clerk, fetches available clouds, passes both as props to client `<AppShell>`.
- `app/app/[org]/(sections)/layout.tsx` — optional inner layout if any group of sections needs shared chrome (e.g. settings inner-nav). Not built in this sub-project.

### 7.3 The `/api/me/preferences` endpoint

`POST` body: `{ railCollapsed?: boolean }`.
Updates Clerk `user.publicMetadata`. Returns 204.
Auth: signed in only. No org scope.

### 7.4 The `/api/orgs/[orgId]/clouds` endpoint

`GET` → `{ clouds: ("aws" | "azure" | "gcp")[] }`.
Source: query `accounts` table grouped by `cloud`. Cached 30s.

## 10. State variants per shell screen

The shell itself has 4 distinct render states:

| State | Trigger | Visual |
|---|---|---|
| **Booting** | Auth resolved, org context still loading | Skeleton rail (no items, just shape) + skeleton topbar + skeleton main |
| **No connected cloud** | Org exists but no AWS/Azure/GCP connected | Renders rail + topbar normally, main is the welcome wizard |
| **Permission denied** | Signed in but not a member of the URL's org | Redirect to `/orgs?error=not-member` (handled in middleware, never renders shell) |
| **Connected, ready** | Normal | Full chrome |

## 11. Performance contracts

- **First paint of shell ≤ 100ms** after route ready. Skeleton must show within one frame.
- **Rail collapse animation ≤ 320ms.** No layout shift in main content.
- **Command bar open ≤ 150ms** including focus.
- **Resource/finding search debounce: exactly 200ms.** Faster = wasted requests; slower = feels laggy.
- **Org switch must be perceived ≤ 500ms** even on slow networks. Loading state shown immediately on click.

## 12. Out of scope (explicitly)

- Notifications inbox (deferred to wave 5+).
- In-app messaging / Intercom / chat widget (deferred forever — support is email + docs).
- A11y contrast modes beyond the default (the design system already meets WCAG AA on the dark palette).
- Sub-nav within sections (settings inner-nav, integrations inner-nav). Each section that needs one designs it later as part of its own sub-project.
- Tab-strip for `?tab=*` overview tabs — that's part of the overview screen, not the shell.
- Profile pictures upload UI — Clerk's component handles it inline in `/me`.

---

## 13. Anti-goals

- No animated rail labels appearing word-by-word.
- No glow / bloom / parallax effects on the rail.
- No "fun" easter-egg shortcuts.
- No A/B testing of the shell layout — engineers hate moving navigation.
- No tooltips that say obvious things ("Click to open Settings").

## 14. Open questions (flag, don't block)

1. **Org sigil color algorithm** — `hash(orgId) % palette`? Or admin-selectable color? Default to hash for now; allow customization in `/app/[org]/settings/general` in wave 4.
2. **Clerk multi-org race condition on switch** — when user switches active org via our dropdown, Clerk's hook updates after router pushes. We must `await setActive(newOrgId)` before `router.push`. Confirm the API is sync-await-able. (Clerk docs say yes.)
3. **Help link target** — `/docs` for now. Replace with in-app help popover if Intercom-style support is ever requested (we said never; leave the link).
4. **Keyboard shortcut conflicts with browser** — `Ctrl+K` on Linux Chrome opens address bar. We intercept; confirm preventDefault works in our middleware.
5. **Mobile drawer gesture** — swipe-from-left to open. Worth wiring Radix Sheet's gesture support? Defer to mobile-polish sub-project.

## 15. What this unlocks

When the shell ships:
- Every subsequent app screen is a single page component that renders inside `<main>`.
- New screen authors do not write nav, do not write breadcrumb logic, do not register keyboard shortcuts beyond their own scope.
- Org and cloud context are available everywhere via `useOrg()` / `useCloud()`.
- ⌘K becomes the universal "where am I going next" surface — every new feature gets one Action and zero new chrome.
- Wave 1 ("end-to-end thinnest journey") can begin: sign-up → org-create → welcome wizard → first finding inside the shell.

A new engineer can write this and it works:

```tsx
// app/app/[org]/findings/page.tsx
export default function FindingsPage() {
  const { org } = useOrg();
  const { active: cloud } = useCloud();
  return <main>Findings for {org.slug} on {cloud}</main>;
}
```

…and they see:
- The full shell chrome (rail collapsed by default, topbar with org + breadcrumbs + ⌘K + cloud switcher + avatar)
- `aria-current="page"` correctly on the Findings rail item
- `⌘K` opens, types "findings" → arrow-down → enter → still on this page (because we're already there)
- `[` toggles the rail, preference persists across reload
- Switching org via the dropdown navigates to `/app/<other>/findings`

That's the bar.
